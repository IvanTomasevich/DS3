Hay cuatro tipos de rutinas almacenadas:
  • Funciones: Se pueden invocar desde cualquier query.
  • Procedimientos: Se pueden invocar desde cualquier aplicación que tenga acceso a la BD.
  • Disparadores (triggers): Se ejecutan automáticamente después de haberse ejecutado un INSERT, UPDATE o DELETE a una cierta tabla
  • Eventos: Se ejecutan automáticamente en un momento agendado.
En esta clase, nos ocuparemos de los dos últimos

Bucles

Bucle WHILE

WHILE condicion DO
    código SQL;
END WHILE;

Se repite mientras la condición sea verdadera

Bucle REPEAT

REPEAT
    código SQL;
UNTIL condicion END REPEAT;

Se repite mientras la condición sea falsa

Bucle LOOP

ejemplo : LOOP
    código SQL;

        LEAVE ejemplo;
    END IF;

CURSORES

Son índices que permiten recorrer el resultado de un SELECT.

Lo explicaremos con un ejemplo:

Realizaremos un procedimiento que defina como NULL todas las fechas de nacimiento futuras

CREATE PROCEDURE verifica_fn()
BEGIN 
  DECLARE contador INT UNSIGNED DEFAULT 0; 
  DECLARE ultima_fila BOOLEAN DEFAULT FALSE; 
  DECLARE numero_var INT UNSIGNED; 
  -- Declaramos un cursor para recorrer el SELECT:
  DECLARE anula_fecha CURSOR FOR 
    SELECT numero FROM ciclistas WHERE fecha_nac > CURDATE(); 
  -- Declaramos una acción para cuando no haya más filas(1329)
  DECLARE CONTINUE HANDLER FOR NOT FOUND 
    SET ultima_fila = TRUE;
  -- Abrimos el cursor:  
  OPEN anula_fecha; 
  -- Tomamos el primer resultado del cursor:
  FETCH anula_fecha INTO numero_var;

  WHILE ultima_fila = FALSE DO 
    UPDATE ciclistas SET fecha_nac = NULL 
        WHERE numero = numero_var; 
    SET contador = contador + 1; 
    -- Tomamos el siguiente resultado del cursor:
    FETCH anula_fecha INTO numero_var; 
  END WHILE; 
  -- Cerramos el cursor:
  CLOSE anula_fecha; 
  SELECT CONCAT("Cilistas con fecha de nacimiento futura: ",
                 contador) as resultado; 
END

Algunos códigos de error

  • 1329: Ocurre al intentar obtener datos de una fila que no existe.
  • 1062: Ocurre cuando se intenta insertar NULL en una columna que no acepta valores nulos.
  • 1216: Ocurre cuando un programa intenta añadir o modificar un registro violando una restricción de clave foránea (foreign key).
  • 1217: Ocurre cuando se intenta eliminar un registro violando una restricción de clave foránea (foreign key)
  • NOT FOUND: Ocurre cuando se intenta usar FETCH o SELECT para obtener datos, pero no se obtiene ningún dato.
  • SQLEXCEPTION: Ocurre cuando la BD devuelve un error
  • SQLWARNING: Ocurre cuando la BD devuelve un error o una advertencia (warning).
